# TDSC 待 PI 与统计师确认清单

本清单基于以下文件的逐条比对形成：

- 正式研究方案 V2.0
- 标准模板重组精修稿
- SAP V1.0
- 知情同意书 V2.1
- 关键 SOP：SOP02、SOP05、SOP11、SOP13、SOP15
- EDC 设置事项中的 NRES 建议文档

本清单只列出当前最可能影响方案一致性、分中心培训、EDC 建库或统计分析口径的事项。段落编号为本次本地提取时的顺序号，用于内部核对定位。

## 一、建议优先拍板的高优先级事项

### 1. 队列A非劣效界值冲突

- 研究方案写法：队列A `δA = 10%`。
- 证据：
- 研究方案[91]：“非劣效界值δA=10%……”
- 研究方案[3007]：“非劣效界值：δA = 10%。”
- SAP 写法：队列A `δA = 8%`。
- 证据：
- SAP[97]：“非劣效界值δA=8%……”
- 需要确认的问题：
- 队列A最终锁定的非劣效界值到底是 `8%` 还是 `10%`。
- 若改为 `8%`，是否需要同步修订正式研究方案、培训稿、样本量说明和伦理材料。
- 影响：
- 直接影响样本量依据、主要检验结论和注册/伦理一致性。

### 2. 队列A把握度（Power）冲突

- 研究方案写法：队列A Power 约 `92%`。
- 证据：
- 研究方案[91]：“Power 约92%……”
- SAP 写法：队列A Power `80%`。
- 证据：
- SAP[97]：“功效80%……”
- 需要确认的问题：
- 队列A样本量依据到底按 `92%` 还是 `80%` 设计。
- 如果 SAP 为最终版本，方案正文是否需要回改。
- 影响：
- 直接影响统计学论证完整性，属于送审时很容易被问到的冲突。

### 3. 队列B显著性水平/置信区间口径存在冲突风险

- 研究方案写法：队列B `α=0.05`。
- 证据：
- 研究方案[93]：“α=0.05；Power 80%……”
- SAP 写法：队列B `单侧α=0.025`，对应 `双侧95%CI`。
- 证据：
- SAP[99]：“单侧α=0.025……”
- SAP[107]：“使用双侧95% CI……”
- 需要确认的问题：
- 队列B最终采用的表述是“单侧α=0.025”还是“总α=0.05”。
- 若两种说法本意相同，建议在正式版中统一成一种监管友好的表达。
- 影响：
- 会影响统计方法章节、SAP、CSR 和稽查时的口径统一。

### 4. 知情同意书将“18个月”写成主要终点评估，和方案/SAP 冲突

- 方案和 SAP 写法：主要终点为停药后 `12个月（PT-M12）`。
- 证据：
- 研究方案[83]、[142]
- SAP[11]、[72]
- SOP11[17]、[34]
- 知情同意书写法：`18个月（主要终点评估）`。
- 证据：
- 知情同意书[50]：“……12个月、18个月（主要终点评估）与停药后24个月。”
- 需要确认的问题：
- 主要终点评估应明确为 `PT-M12`，18个月应是什么定位。
- ICF 是否需要立即修订，以免受试者沟通和伦理版本出现硬冲突。
- 影响：
- 属于高风险伦理一致性问题，也会直接误导分中心培训。

### 5. 停药后随访窗口在 SAP/SOP11 与 SOP13/SOP15 之间不一致

- SAP/SOP11 写法：PT-M12、PT-M24 允许窗口 `±30天`。
- 证据：
- SAP[17]：“PT-M12/PT-M24±30天”
- SOP11[17]：“PT-M12……允许窗口±30天”
- SOP13/SOP15 写法：停药后随访 FV 窗口为 `±14天`。
- 证据：
- SOP13[7]：“FV1–FV7 为相对 EOT 的时间窗（±14天）”
- SOP13[73]-[76]：“+1月±14、+3月±14、+6月±14、+9月±14”
- SOP15[61]：“停药后随访（FV）以……EOT日期为锚点计算（±14天窗口）”
- 需要确认的问题：
- `PT-M12/PT-M24` 这两个主要分析窗口最终按 `±30天` 还是 `±14天`。
- 是否允许一般 FV 仍按 ±14天，但 `PT-M12/PT-M24` 特殊放宽到 ±30天。
- 影响：
- 直接影响主要终点可评价性、失访判定、EDC edit check 和中心排程。

### 6. M6 Hard Gate 访视窗口不一致

- SOP02 写法：M6 为 `W24，D168±7`。
- 证据：
- SOP02[20]：“M6（W24，D168±7）随机化前……”
- SAP 写法：M6/M9 访视窗口 `±14天`。
- 证据：
- SAP[17]：“M6/M9±14天”
- 需要确认的问题：
- Hard Gate 判定用的 M6 窗口最终是 `±7天` 还是 `±14天`。
- 如果 M6 访视可 ±14天，Hard Gate 所需培养证据的时间锚点如何统一。
- 影响：
- 会直接影响随机资格、EDC 自动判断和现场实际可操作性。

## 二、建议本轮一并拍板的中高优先级事项

### 7. 随机化分层因素在方案与 SOP05 之间不完全一致

- 研究方案写法：建议/预设分层因素为 `中心 + 菌负荷 + 影像范围 + HbA1c`。
- 证据：
- 研究方案[81]、[108]
- SAP 写法：模型中纳入 `基线菌负荷 + 基线影像范围 + 基线HbA1c`，中心视情况处理。
- 证据：
- SAP[104]
- SOP05 写法：
- 队列A“至少按研究中心分层；其他因素按随机化规格书”
- 队列B“强制包含 HbA1c 分层；其余因素按规格书”
- 证据：
- SOP05[72]-[73]、[118]、[166]
- 需要确认的问题：
- 队列A和队列B最终锁定的随机化分层因素分别是什么。
- `HbA1c` 是否为两队列都强制纳入的分层因素。
- 影响：
- 影响 IRT 设置、统计调整变量和培训时的统一表述。

### 8. “Hard Gate 未达标”与“Hard Gate 无法评价”是否分开分析尚未锁死

- 方案/SOP02/ICF 的主口径更接近：无法评价者按未达标，不进入随机化。
- 证据：
- 知情同意书[42]-[46]
- SOP11[60]-[61]
- EDC NRES 建议文档则单列三种状态：
- 通过 Hard Gate
- 未通过 Hard Gate
- 无法完成/无法评价 Hard Gate
- 证据：
- EDC_NRES[9]-[10]
- 需要确认的问题：
- 统计分析和 EDC 主状态值是否区分“未通过”和“无法评价”。
- 如果区分，CSR 与 SAP 是否要预设这类描述性分析。
- 影响：
- 影响 EDC 结构、NRES 建模和队列B描述性结果输出。

### 9. SAP 中关键访视列表较简，和方案/SOP13 的全访视主表不完全对称

- 正式方案列出：W2、W4、W8、W12、W16、W20、M6、W28、W32、M9、PT-M12、PT-M24，且还有 18 个月访视。
- 证据：
- 研究方案[612]
- SAP 概览中只突出：W0、W2、W8、W16、M6、M9、PT-M12、PT-M24。
- 证据：
- SAP[17]
- 需要确认的问题：
- SAP 是否有意只列“关键分析访视”，还是遗漏了若干 protocol-required 访视。
- 若只是简写，是否需要在 SAP 中注明“其余访视见 visit schedule/SOP13”。
- 影响：
- 对统计程序影响不一定大，但对中心培训和数据预期会造成歧义。

### 10. 安全性分析时间锚点建议再次明确，尤其是队列B预随机阶段

- SAP 明确主要安全性终点时间起点为“随机化日期”。
- 证据：
- SAP[89]
- 研究方案执行层容易被理解为“整个治疗期和随访期安全性”。
- 证据：
- 研究方案[144]-[153] 存在较宏观表述
- 重组精修稿[106]、[243] 已向 SAP 靠拢
- 需要确认的问题：
- 队列B 在 M0-M6 统一治疗阶段发生的 AE/SAE，在主安全性分析中的定位是什么。
- 是否需要把“随机前队列B安全性”作为单独预设描述性表格。
- 影响：
- 影响 SAP 输出、CSR 结构和培训时对 AE 重点时窗的解释。

## 三、建议作为培训/建库补充说明但不必阻塞主版本的事项

### 11. NRES 是否上升为正式分析对象

- 当前 EDC 设置事项已提出较完整的 NRES 字段框架。
- 证据：
- EDC_NRES[6]-[18]
- 方案和 SAP 已承认未达标者继续治疗并做描述性分析，但未完全固定字段层级和输出模板。
- 证据：
- 研究方案[79]、[93]
- SAP[15]
- 需要确认的问题：
- 是否正式定义 `NRES（非随机化延续亚队列）` 作为队列B的固定描述性分析对象。
- 是否在 SAP TLF 中增加专章。

### 12. HbA1c 阈值仍属于“锁定前变量”

- 研究方案明确提到 HbA1c 分层“阈值以 SAP/SOP 锁定”。
- 证据：
- 研究方案[81]
- 但当前抽取内容中未见统一阈值正文。
- 需要确认的问题：
- HbA1c 分层阈值的最终取值和适用队列。
- 是否已在随机化规格书中锁定。

## 五、我的总体判断

当前冲突里，最优先需要立刻拍板的是 6 项：

- 队列A `δA`
- 队列A power
- 队列B α/CI 口径
- ICF 的“18个月主要终点评估”错误
- PT-M12/PT-M24 访视窗口
- M6 Hard Gate 窗口

如果这 6 项不先统一，后续分中心培训、EDC 建库、统计程序和伦理材料都会继续出现版本漂移。
