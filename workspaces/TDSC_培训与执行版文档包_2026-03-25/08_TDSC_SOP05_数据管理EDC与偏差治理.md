# TDSC SOP05：数据管理、EDC 与偏差治理

## 1. 目的

统一 EDC 数据采集、Query、数据清理、锁库、方案偏差和治理架构。

## 2. 适用范围

适用于 CRC、数据管理员、项目经理、监查员和研究中心负责人。

## 3. 数据管理原则

- 源数据真实、完整、可追溯。
- eCRF 录入及时。
- 关键变量优先清理。
- 锁库前确保主要终点可评价。

## 4. EDC 录入重点

- 资格、分层、随机化。
- 实际治疗暴露。
- 培养和影像关键结果。
- EOT 日期和停药后事件。
- AE/SAE/AESI。
- 队列B Hard Gate 达标状态和未达标原因。

## 5. Query 管理

- Query 应分级处理。
- 关键终点和关键安全性 Query 优先关闭。
- 所有关闭动作保留审计痕迹。

## 6. 方案偏差管理

- 发现偏差后及时记录、分级和分析根因。
- 重大偏差应启动 CAPA。
- 同类偏差重复出现时，应触发中心级再培训。

## 7. 锁库前核查

- 主要终点相关日期和事件闭环。
- 随机化和治疗路径一致性。
- EOT、停药后访视和结局状态一致性。
- 未达标随机化和误入组情况复核。

## 8. 组织职责

- 协调中心负责总体规则和版本控制。
- 中心 PI 对本中心数据质量负责。
- PMO 负责培训、会议和跨团队协调。
- DSMB、CEC 和其他独立委员会按各自章程运行。
