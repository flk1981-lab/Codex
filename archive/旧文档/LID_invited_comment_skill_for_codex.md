# LANCET ID invited Comment 写作 Skill（供 Codex 学习）

> 目的：指导 AI / Codex 为 **The Lancet Infectious Diseases（LID）invited Comment** 生成接近可投稿水准的稿件与修改策略。  
> 适用对象：以临床研究、感染病学、结核病等主题为主的 commissioned / invited Comment。  
> 使用原则：**先服从当次编辑邀请与当期作者说明；本 Skill 作为高质量默认框架。**

## 一、先确认的“硬约束”

在开始写作前，先核对当次邀请邮件、投稿系统和最新官方作者说明。当前官方公开信息提示：

- LID 的 **Commentaries** 一般为 **不超过 750 词、最多 10 条参考文献**。  
- Lancet 体系要求作者申报 **利益冲突**、**资金来源及其作用**；使用生成式 AI 参与写作时，需在文末加入 **单独的 AI 声明**。  
- 参考文献通常按 **Vancouver** 风格整理。  
- 稿件必须为作者本人作品，**未在他处发表，也未同时投往其他期刊**。  
- 具体要求会随时间更新，投稿前必须再次核对最新网页、邀请邮件与系统表单。

## 二、对 LID Comment 的根本定位

### 1. Comment 不是综述
不要把 invited Comment 写成“把背景讲全、把机制讲满、把文献讲遍”的综述。

### 2. Comment 不是导读
不要只是把原始论文的设计、结果和结论重新说一遍。

### 3. Comment 不是站台文
不要因为认可该研究方向，就替该研究“争取地位”、替该领域“做宣传”。

### 4. Comment 的本质
**Comment 的本质，是为一篇新论文分配它应得的证据级别。**

换句话说，作者最重要的任务不是“写得丰富”，而是：

- 说清楚它 **真正证明了什么**
- 说清楚它 **还没有证明什么**
- 指出它在本领域里 **应被放在什么位置**
- 告诉读者 **下一步真正关键的问题是什么**

## 三、写作时必须建立的观念 / 高度

### 1. 先划边界，再谈意义
任何宏大意义、平台意义、未来方向，都要建立在当前论文真实证据边界之上。  
**没有边界的意义，不可信。**

### 2. 终点层级决定论证高度
评价临床研究时，先判断其主要终点属于哪一层：

- 替代终点 / 生物学信号
- 中期临床终点
- 长期、患者真正关心的临床结局

谁先抓住终点层级，谁就抓住了 Comment 的制高点。

### 3. 用编辑视角，而不是项目负责人的视角
高水平 Comment 需要像“证据守门人”，而不是“方向推动者”。

- 少一点推动
- 少一点申辩
- 少一点替研究争取空间
- 多一点证据分级
- 多一点边界限定
- 多一点冷静裁决

### 4. 真正的权威，不靠说得远，而靠收得准
对顶刊 Comment 而言，**收住**往往比**展开**更高级。

## 四、LID Comment 的推荐结构（默认 5–6 段）

> 目标：在 750 词左右完成“背景—试验判读—边界—意义—安全性/实施性—下一步”闭环。

### 第 1 段：背景 + 立题
任务：

- 用 1–2 句建立临床问题或研发问题
- 迅速说明这篇论文为什么值得评论
- 不要一上来就给出过强结论

写法要点：

- 先交代“大问题”
- 再点出“这篇研究提出了一个新的具体问题”
- 结尾引出本文核心判断方向

### 第 2 段：试验设计与关键结果
任务：

- 交代研究类型、分期、场景、人群、比较对象
- 明确主要终点
- 给出 **关键数字结果**
- 立即点出应如何解读这些结果

写法要点：

- 不只写“结果不错”，而要写出关键数值
- 不只写“达非劣效”，而要说清楚终点是什么
- 说明这是 **proof-of-concept**、还是 **practice-changing** 证据

### 第 3 段：真正关键的 distinction
任务：

- 明确指出：当前研究最重要的“证据边界”是什么
- 必要时用历史试验做对照（如 REMoxTB 之类）

这一段通常要回答：

- **“短期积极信号”不等于什么？**
- **“当前终点”与“真正临床关心的终点”有什么差别？**
- **下一步真正该验证的是什么？**

### 第 4 段：研究优点（strengths）
任务：

- 平衡叙事，不能只批判
- 用 1 段简明指出研究设计或研发路径上的亮点

可写内容：

- 创新机制 / 新组合
- 多中心随机设计
- 设有 active comparator
- 剂量探索合理
- 为后续确证性研究提供了可信信号

### 第 5 段：更广意义，但必须降调
任务：

- 讨论 broader significance
- 但必须立刻加上边界限定

标准写法：

- present data remain confined to ...
- primary interpretation should remain there
- may help inform ...
- should not be taken as establishing ...
- if confirmed ...

核心原则：  
**可以升华，但必须轻、短、可撤回。**

### 第 6 段：安全性 / 可实施性 / 下一步
任务：

- 安全性不能泛泛而谈，要尽量给数字
- 讨论可实施性、监测负担、长期结局
- 用一句话收束“下一步真正需要什么研究”

常见收尾方向：

- durability
- implementability
- clinically meaningful long-term endpoints
- larger and broader populations
- confirmatory evaluation

## 五、句子级写作技巧

### 1. 多用“判断句”，少用“说明句”
优先使用这类句型：

- The findings should be interpreted with caution.
- The key distinction is between ... and ...
- The next question is whether ...
- These results provide a credible signal, but ...
- What this study may contribute, if confirmed, is ...

避免全文都在做“论文说明书”。

### 2. 一切正面评价都要带刹车
优先使用：

- may
- suggests
- supports
- signal
- may contribute
- may help inform
- merits further evaluation
- if confirmed

慎用：

- establishes
- demonstrates
- defines
- confirms
- transformative
- platform（除非证据确实足够）

### 3. 历史对照必须服务于当前判断
引用历史试验，不是为了显示熟悉文献，而是为了完成一句判断：

- 过去哪里翻过车？
- 这意味着当前研究应如何被理解？
- 哪个终点 / 哪个研发目标更难？

### 4. 每段只承担一个主任务
不要让一段同时承担：

- 背景
- 设计
- 机制
- 安全性
- 宏观展望

段落功能必须清楚，否则 750 词内会很乱。

## 六、最常见的错误

### 1. 写成 mini-review
背景太长，机制太满，导致评论对象被淹没。

### 2. 只下结论，不给数字
Comment 不能只有“锋利判断”，还要有足够的数值支撑。

### 3. 用当前试验去支持超出证据边界的大命题
例如：

- 直接写成跨人群平台已建立
- 直接暗示会改变实践
- 用药敏数据替耐药人群背书

### 4. 反复重复同一个判断
同一层意思只需要最强的一句，不要连续两三句重复发力。

### 5. 安全性写得过轻
如果原文有 serious TEAEs、grade 3/4 AEs、QT 风险、监测要求，就必须诚实呈现，不能只写“总体可接受”。

### 6. 结尾过满
最差的收尾方式是：

- 再总结一次全文
- 再夸一遍研究
- 再展望一遍未来

最好的结尾是：  
**一句裁决 + 下一步研究问题。**

## 七、LID invited Comment 的修订工作流

### 第 1 轮：先完成“主判断”
先用一句话写出全文核心判断。若一句话写不出来，说明思路未定。

模板：
> This study provides a credible early signal for X, but it does not yet establish Y.

### 第 2 轮：核对原始论文
逐条核对以下信息：

- 研究设计
- 样本量
- 随机分组
- 主要终点
- 非劣效界值 / alpha / CI
- 关键数字结果
- 长期结局
- 安全性结果
- 作者讨论中提到的 ongoing trials

### 第 3 轮：清理“多余的好内容”
删除所有虽然正确、但对这篇 Comment **不是必须** 的句子。

判断标准：
> 这句话是不是“高明”不重要；重要的是，它是否“必不可少”。

### 第 4 轮：平衡叙事
确保正文里同时有：

- 主判断
- 研究优点
- 研究边界
- 下一步方向

### 第 5 轮：投稿元件检查
必须单独核对：

- 标题是否克制、准确
- 作者贡献
- 利益冲突
- Funding / role of funder
- AI declaration
- 通讯作者信息
- 参考文献格式
- 投稿系统问答与正文一致

## 八、可直接执行的 Codex 规则

把下面这些规则当成默认行为：

1. **永远先写证据边界，再写意义。**
2. **永远先看主要终点属于哪一层。**
3. **永远至少给出关键数值，而不是只写定性判断。**
4. **永远把“strengths”单独处理，避免一边倒。**
5. **永远把 broader significance 降调处理。**
6. **永远把 safety 与 implementability 放入最终裁决。**
7. **永远在提交前检查 AI、COI、Funding、author forms、reference style。**
8. **若编辑提出“more balanced”或“more context”，优先扩充背景、设计、优点与数值，而不是去争论措辞。**
9. **若一句话和下一句表达同一层意思，删掉较弱的一句。**
10. **默认目标不是“写得像作者本人很懂”，而是“写得像编辑愿意刊出”。**

## 九、给 Codex 的标准任务提示词（可直接复用）

### 模式 A：起草整篇 invited Comment
你现在要为 The Lancet Infectious Diseases 起草一篇 invited Comment。请遵守以下原则：
1. 先界定原始论文真正证明了什么、没有证明什么。
2. 不要写成综述，不要写成导读，不要替该研究站台。
3. 全文优先强调主要终点的层级与证据边界。
4. 必须给出关键设计信息和关键数字结果。
5. 至少单独安排一段 strengths，一段 broader significance with caution，一段 safety/implementability/next step。
6. 全文语气应克制，优先使用 may / suggests / signal / if confirmed / merits further evaluation。
7. 避免重复表达同一判断。
8. 默认按 5–6 段、约 750 词、最多 10 条参考文献来写；若最新作者说明不同，以最新说明为准。
9. 稿件末尾补充：AI statement、declaration of interests、作者贡献、funding（如适用）。

### 模式 B：审阅并重写已有 Comment
请以 LID 资深编辑的标准审阅这篇 invited Comment，并完成以下任务：
1. 用一句话概括本文主判断是否成立。
2. 找出所有超出原始论文证据边界的句子。
3. 找出所有缺少关键数字支撑的判断句。
4. 判断结构是否符合“背景—设计/结果—distinction—strengths—broader significance—safety/next step”。
5. 给出可直接替换的英文句子，而不是空泛建议。
6. 修改目标不是“更好看”，而是“更平衡、更可信、更接近可刊发”。

## 十、最后一句总纲

**LID invited Comment 的最高目标，不是把一篇论文讲明白，而是把它放到它应在的位置上。**
