
LANCET ID invited Comment 写作 Skill
供 Codex 学习与执行的高质量默认框架
目的：指导 AI / Codex 为 The Lancet Infectious Diseases（LID）invited Comment 生成接近可投稿水准的稿件与修改策略。
定位：先服从当次编辑邀请与最新作者说明；本 Skill 作为高质量默认框架。
总纲：不是“介绍一篇论文”，而是“为一篇论文分配它应得的证据级别”。
一、先确认的硬约束
开始写作前，必须核对当次邀请邮件、投稿系统与最新官方作者说明。默认可据此把握以下基线：
• LID 的 Commentaries 一般为不超过 750 词、最多 10 条参考文献。
• Lancet 体系要求申报利益冲突、资金来源及其作用；若使用生成式 AI 协助写作，需加入单独 AI 声明。
• 参考文献通常按 Vancouver 风格整理。
• 稿件必须为作者本人作品，未在他处发表，也未同时投往其他期刊。
• 凡涉及具体字数、图表数、表单、签名件、通讯作者信息等，均以最新官方网页和当次系统要求为准。
二、对 LID Comment 的根本定位
1. Comment 不是综述
不要把 invited Comment 写成“把背景讲全、把机制讲满、把文献讲遍”的综述。
2. Comment 不是导读
不要只是把原始论文的设计、结果和结论重新说一遍。
3. Comment 不是站台文
不要因为认可该研究方向，就替该研究“争取地位”、替该领域“做宣传”。
4. Comment 的本质
Comment 的本质，是为一篇新论文分配它应得的证据级别。作者最重要的任务不是“写得丰富”，而是说清楚它真正证明了什么、还没有证明什么、应被放在什么位置，以及下一步关键问题是什么。
三、写作时必须建立的观念 / 高度
1. 先划边界，再谈意义
任何宏大意义、平台意义、未来方向，都必须建立在当前论文真实证据边界之上。没有边界的意义，不可信。
2. 终点层级决定论证高度
先判断主要终点属于替代终点、生物学信号、中期临床终点，还是长期且患者真正关心的临床结局。谁先抓住终点层级，谁就抓住 Comment 的制高点。
3. 用编辑视角，而不是项目负责人的视角
高水平 Comment 要像证据守门人，而不是方向推动者：少一点推动与申辩，多一点证据分级、边界限定与冷静裁决。
4. 真正的权威，不靠说得远，而靠收得准
对顶刊 Comment 而言，收住往往比展开更高级。删掉诱人但不必要的内容，通常比增加更多正确内容更重要。
四、LID Comment 的推荐结构（默认 5–6 段）
目标是在约 750 词内完成“背景—试验判读—边界—优点—更广意义—安全性/下一步”的闭环。
段落
功能
必须完成的任务
第1段
背景 + 立题
用 1–2 句建立临床或研发问题；说明这篇论文为什么值得评论；不要一上来就给出过强结论。
第2段
设计 + 关键结果
交代研究类型、分期、场景、人群、比较对象和主要终点；给出关键数值结果，并立即点出应如何解读。
第3段
真正关键的 distinction
明确指出当前研究最重要的证据边界；必要时用历史试验做对照；说明下一步真正该验证的终点。
第4段
研究优点
单独写 strengths，保证叙事平衡；可写创新机制、随机设计、active comparator、剂量探索等。
第5段
更广意义，但必须降调
可以讨论 broader significance，但必须马上加边界限定，使用 may / if confirmed / should not be taken as 等表达。
第6段
安全性 / 可实施性 / 下一步
安全性尽量给数字；讨论监测负担、长期结局和 implementability；用一句裁决收尾。
五、句子级写作技巧
1. 多用判断句，少用说明句
优先写 “The findings should be interpreted with caution.” “The key distinction is between … and …” “The next question is whether …” 这类判断句，而不是整篇都在做说明书。
2. 一切正面评价都要带刹车
优先使用 may、suggests、supports、signal、may contribute、may help inform、merits further evaluation、if confirmed；慎用 establishes、demonstrates、defines、confirms、transformative、platform。
3. 历史对照必须服务于当前判断
引用历史试验不是为了显示熟悉文献，而是为了回答：过去哪里翻过车？这意味着当前研究该如何被理解？哪个研发目标更难？
4. 每段只承担一个主任务
在 750 词内，段落功能必须清楚；不要让一段同时承担背景、设计、机制、安全性和宏观展望。
六、最常见的错误
• 写成 mini-review：背景太长、机制太满，评论对象被淹没。
• 只下结论，不给数字：缺少关键数值会削弱可信度。
• 用当前试验去支持超出证据边界的大命题：例如直接写成跨人群平台已建立或会改变实践。
• 重复表达同一判断：同一层意思只需要最强的一句。
• 安全性写得过轻：若原文有 serious TEAEs、grade 3/4 AEs、QT 风险或监测要求，必须诚实呈现。
• 结尾过满：不要再总结一遍全文、再夸一遍研究；最好的结尾是一句裁决 + 下一步研究问题。
七、LID invited Comment 的修订工作流
第 1 轮：先完成主判断
先用一句话写出全文核心判断。若一句话写不出来，说明思路未定。模板：This study provides a credible early signal for X, but it does not yet establish Y.
第 2 轮：核对原始论文
逐条核对研究设计、样本量、随机分组、主要终点、非劣效界值 / alpha / CI、关键数字结果、长期结局、安全性结果、ongoing trials。
第 3 轮：清理“多余的好内容”
删除所有虽然正确、但对这篇 Comment 不是必须的句子。判断标准：这句话是不是高明不重要；重要的是，它是否必不可少。
第 4 轮：平衡叙事
确保正文里同时有主判断、研究优点、研究边界和下一步方向。
第 5 轮：投稿元件检查
单独核对标题、作者贡献、利益冲突、Funding / role of funder、AI declaration、通讯作者信息、参考文献格式以及投稿系统问答与正文是否一致。
八、可直接执行的 Codex 规则
• 永远先写证据边界，再写意义。
• 永远先看主要终点属于哪一层。
• 永远至少给出关键数值，而不是只写定性判断。
• 永远把 strengths 单独处理，避免一边倒。
• 永远把 broader significance 降调处理。
• 永远把 safety 与 implementability 放入最终裁决。
• 永远在提交前检查 AI、COI、Funding、author forms、reference style。
• 若编辑提出 more balanced 或 more context，优先扩充背景、设计、优点与数值，而不是去争论措辞。
• 若一句话和下一句表达同一层意思，删掉较弱的一句。
• 默认目标不是“写得像作者本人很懂”，而是“写得像编辑愿意刊出”。
九、给 Codex 的标准任务提示词（可直接复用）
模式 A：起草整篇 invited Comment
你现在要为 The Lancet Infectious Diseases 起草一篇 invited Comment。先界定原始论文真正证明了什么、没有证明什么；不要写成综述、导读或站台文；必须突出主要终点层级与证据边界，给出关键设计信息和关键数字结果，至少单独安排一段 strengths、一段 broader significance with caution、一段 safety/implementability/next step。全文语气必须克制，默认按 5–6 段、约 750 词、最多 10 条参考文献来写；若最新作者说明不同，以最新说明为准。
模式 B：审阅并重写已有 Comment
请以 LID 资深编辑的标准审阅这篇 invited Comment：先用一句话概括本文主判断是否成立；找出所有超出原始论文证据边界的句子；找出所有缺少关键数字支撑的判断句；判断结构是否符合“背景—设计/结果—distinction—strengths—broader significance—safety/next step”；最后给出可直接替换的英文句子，而不是空泛建议。修改目标不是“更好看”，而是“更平衡、更可信、更接近可刊发”。
十、最后一句总纲
LID invited Comment 的最高目标，不是把一篇论文讲明白，而是把它放到它应在的位置上。
