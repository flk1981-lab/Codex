# 临床研究方案制定与修改规则（含价值定位、科学性与可行性原则）

**供人工审阅与 Codex 自动化改写使用**  
**版本：V2.0｜日期：2026-03-25**

## 0. 顶层说明

- **规则优先级**：法律法规/伦理与注册要求 > 资助方/申办方/机构模板 > 研究类型专门规范 > 通用 protocol 规范 > 期刊投稿偏好。
- **使用场景**：适用于临床试验、单臂或非随机干预研究、观察性研究、诊断研究、预测模型/AI 研究，以及需要向伦理、基金、注册平台或期刊提交的研究方案。
- **核心原则**：方案不仅要“写完整”，更要“值得做、做得成、做出来可信、做完后能发表并能影响决策”。

## 1. 文档目的与使用方式

- 本规则包用于帮助合作单位临床医生起草、修改、审阅和标准化临床研究方案（protocol）。
- 使用时必须同时检查两层内容：第一层是格式与合规，确保 protocol 在伦理、注册、执行和投稿层面不缺项；第二层是立题质量，判断该研究是否有明确价值定位、科学问题、可执行路径和可信的证据产出。
- 对于 Codex 或其他自动化模块，本文件既是写作规范，也是审稿清单。自动化模块不得只做语言润色，而必须同步审查研究价值、科学性、可行性和伦理性。

## 2. 修改研究方案时的总原则

- 任何修改都应优先维护研究问题、设计、终点、样本量、执行路径、统计分析和伦理承诺之间的一致性，避免只改文字不改逻辑。
- Protocol 不是基金标书的背景综述，也不是论文的引言扩展版。它必须像“可执行作业书”一样，足以让研究团队、伦理委员会、数据管理人员和审稿人理解研究为何开展、如何实施、如何保证质量、如何解释结果。
- 不能用空泛词替代具体定义。凡涉及研究对象、暴露/干预、终点、随访窗口、分析集、缺失数据处理、安全性上报、终止规则、样本量假设、亚组定义等，均应尽量量化、预先界定并写成可执行语言。
- 如存在研究者尚未确定的信息，必须显式标注为“[待补充]”或“[需PI确认]”，禁止自动补造机构名称、入组率、效应量、失访率、经费、注册号、伦理批件号或未被提供的先导数据。

## 3. 研究方案的价值定位原则：先回答‘为什么值得做’

- 研究价值定位必须先于方法学包装。一个 protocol 的首要任务不是显示写得多规范，而是说清楚：现有证据缺口是什么，这个缺口为什么重要，研究完成后将改变什么决策。
- 建议用一句话写清研究定位：在什么人群/场景中，针对什么关键临床或公共卫生问题，利用什么设计，解决哪一类未决策的证据空白。
- 价值定位至少要同时回答四个问题：一是疾病负担或临床痛点是否重要；二是现有证据为何不足；三是本研究相比既有研究新增了什么；四是本研究结果将影响谁的决策（临床、指南、政策、患者、支付方、后续试验）。
- 对合作单位项目，尤其要强调‘为什么这个团队、这个场景、这个时间点适合做这项研究’，例如病例来源、平台资源、样本检测能力、长期随访条件、真实世界代表性或多中心协作网络。

## 4. FINER 原则及其实务化扩展

- FINER 是研究问题质量判断的经典框架：Feasible（可行）、Interesting（有吸引力/有兴趣）、Novel（有新意）、Ethical（合乎伦理）、Relevant（相关且重要）。
- Feasible：样本来源真实可得，关键变量能测，团队有能力执行，预算与周期基本匹配，随访完成率有现实保障。若研究依赖极低发生率事件、极复杂检测平台、过多中心协调或长期高质量随访，则必须给出先导数据或替代方案。
- Interesting：不仅研究者自己感兴趣，还应当让合作团队、伦理、基金评委、审稿人或潜在读者认为值得投入资源。临床医生发起的研究，应尽量围绕真实诊疗痛点，而不是为做研究而做研究。
- Novel：新意不等于前所未有。真正的新意可来自新问题、新人群、新场景、新终点、新比较、新方法、新机制链条，或对既有证据的重要外部验证。
- Ethical：风险收益比合理，受试者保护措施明确，知情同意可行，隐私与数据安全可控，弱势群体纳入有充分理由，监测与救济安排充分。
- Relevant：研究结果必须能产生实际影响。相关性既包括学术相关性，也包括临床、管理、政策和支付决策相关性。
- 实务中建议把 FINER 扩展为‘FINER+’：在 FINER 之外，再检查 Rigorous（是否方法学严谨）、Actionable（结果能否转化为行动）和 Publishable（完成后是否有明确发表出口）。这是工作中的经验性扩展，不是对经典 FINER 的正式改写。

## 5. 从问题到设计：保证研究问题、目标和方法同向

- 研究问题应先被结构化，再进入方案撰写。常见工具包括 PICOT/PICO（人群、干预/暴露、对照、终点、时间）以及 estimand 思维（目标治疗效应/处理效应究竟是什么）。
- 研究目的必须分层书写：总体目的、主要目的、次要目的、探索性目的。每一个目的都要能够对应到具体终点、测量时间点和分析方法。
- 主要研究问题只能有一个主轴。若同时追求疗效、安全性、机制、经济学、模型开发和实施评价，往往会导致主问题不清、样本量依据混乱、结论分散。此时应明确主次等级，必要时拆分为主研究与附属研究。
- 假设应与设计类型匹配。优效、非劣效、等效、描述性、探索性、预测性和诊断准确性研究的假设结构不同，不能套用同一种写法。

### 5.1 问题类型与设计匹配表

| 研究意图 | 优先设计 | 写作提醒 |
|---|---|---|
| 评价干预效果 | 随机对照试验优先 | 若非随机，应解释为何不能随机，并补充偏倚控制方案 |
| 观察暴露与结局关联 | 前瞻性或回顾性队列 | 写清暴露定义、时间零点、混杂控制 |
| 评价诊断准确性 | 前瞻性连续入组诊断研究优先 | 必须交代 index test、reference standard、盲法和时间间隔 |
| 构建/验证预测模型 | 前瞻性优先，允许高质量回顾性数据 | 必须区分开发与验证，避免仅内部验证就宣称可临床应用 |
| 实施或流程优化 | 实施研究/混合设计/真实世界研究 | 需同时写清效果指标和实施指标 |

## 6. 科学性原则：让结果真实、可解释、可重复

- 科学性的核心不是‘用了高级统计学’，而是研究设计能否真实回答研究问题，并尽可能减少系统误差。
- 研究对象定义应精准：纳入/排除标准、招募来源、筛选流程、基线时间点、研究起始时间零点必须清楚；观察性研究尤其要避免纳入时点不一致造成不朽时间偏倚。
- 干预或暴露定义应具可执行性：谁来给药/实施，剂量或强度如何，允许的合并治疗是什么，依从性如何评估，偏离方案如何记录。
- 对照组必须具有解释力。选择空白对照、标准治疗、历史对照、平行对照或自身前后对照时，都要说明为什么该对照能够支持预期结论。
- 终点必须临床上有意义，且测量方式可标准化。主要终点不宜过多，最好1个；复合终点要说明各组分的临床权重和构成逻辑；安全性终点应明确分级标准与特别关注事件。
- 应尽量预先界定敏感性分析、亚组分析和中期分析，而不是在结果出来后再‘按数据写故事’。
- 统计分析计划应与研究目标一致：描述性研究强调估计与不确定性；比较性研究强调效应量、置信区间和预设假设；模型研究强调开发、验证、校准、判别、临床效用与偏倚风险。

## 7. 样本量与统计分析原则

- 样本量不是‘大概收多少例’，而是对主要研究问题的定量承诺。必须写明：主要终点、目标效应、假设检验类型、显著性水平、把握度、组间分配、事件率或标准差假设、失访/退出膨胀比例，以及假设来源。
- 如果样本量受客观资源限制，也应坦诚写明，并把研究定位调整为探索性、可行性研究、试点研究或描述性研究，避免把能力不足的研究包装成确认性研究。
- 非随机研究不能以‘样本量越多越好’代替设计质量。关键在于暴露定义、混杂控制、时间结构与缺失数据处理。
- 预测模型研究要避免单纯按总样本数拍脑袋。应考虑事件数、模型复杂度、候选变量数、外部验证样本和过拟合控制。
- 临床试验若存在治疗中断、救援治疗、死亡、交叉、停药等 intercurrent events，应在 protocol 中明确这些事件如何被纳入目标效应定义和主分析框架。

## 8. 可行性原则：让方案真的能做成

- 可行性评估至少包含六个方面：招募可行性、执行可行性、测量可行性、随访可行性、团队可行性和预算/周期可行性。
- 招募可行性要有真实依据，例如历史门诊量、住院量、筛查失败率、符合标准比例、中心启动速度、竞争研究情况和患者接受度。
- 执行可行性要检查药物/设备/检测试剂供应、培训、质控、随机分配或分层流程、EDC 建设、样本运输、冷链、影像判读和 SAE 处理链条。
- 测量可行性要求核心变量可稳定获得，且各中心可标准化。凡需高成本检测、中心实验室、长期存储或复杂算法的研究，必须写明替代路径和失败预案。
- 随访可行性要估计失访原因、补访方式、窗口期容忍度、依从性管理、远程随访与结局确认来源。
- 团队可行性要说明 PI、统计师、研究护士、数据管理员、药师、实验室和影像团队的角色分工。多中心研究应明确牵头单位、培训、监查和偏差纠正机制。
- 预算/周期可行性不能只写总经费。应至少识别高成本环节和关键里程碑，避免出现‘方案设计很漂亮，但资源完全支撑不了’的情形。

## 9. 伦理性与受试者保护原则

- 任何 protocol 都必须把受试者安全、权益、尊严和隐私放在中心位置。伦理性不是单独一章，而应贯穿立题、招募、干预、数据管理、结果发布全过程。
- 必须说明潜在获益、潜在风险、不良事件识别与处置、暂停/退出标准、知情同意流程、补偿和救济安排、隐私保护及数据去标识化方法。
- 涉及弱势人群、危重患者、未成年人、孕产妇、认知障碍者或高污名疾病时，应额外说明纳入必要性、附加保护措施和公平性考虑。
- 方案修改若影响受试者风险、终点定义、统计策略或样本量，应说明修订后如何重新伦理备案、版本控制和培训执行。

## 10. Protocol 结构与最少必备内容

1. 封面信息：题目、版本号、日期、申办方/研究者信息、保密属性（如需要）
2. 项目摘要/研究摘要
3. 背景与立题依据：疾病负担、证据缺口、前期依据、研究价值定位
4. 研究目的与假设：总体、主要、次要、探索性
5. 研究设计：类型、框架、流程图、中心设置、随机/盲法/分层（如适用）
6. 研究对象：纳入、排除、退出、终止、筛选流程、招募来源
7. 干预/暴露/检测方法/对照：执行细节、允许与禁止的合并措施
8. 访视与评估：时间窗、检查项目、依从性、样本采集、质控
9. 终点定义：主要、次要、安全性、探索性终点；判定规则；结局确认来源
10. 样本量依据
11. 统计分析计划：分析集、主分析、敏感性分析、亚组分析、缺失数据处理、中期分析（如有）
12. 数据管理与质量控制：EDC、监查、稽查、偏差管理、版本控制
13. 安全性管理：AE/SAE/AESI 定义、上报链条、暂停规则、DSMB（如适用）
14. 伦理学与知情同意
15. 数据安全、隐私与样本/数据共享政策
16. 研究组织与职责分工
17. 时间计划、可行性基础与里程碑
18. 参考文献
19. 附录：知情同意书、CRF/访视表、量表、操作 SOP、流程图、实验室手册等

## 11. 不同研究类型的附加规则

- 随机对照试验：优先按照 SPIRIT 2025 框架组织；写清随机方法、分配隐藏、盲法、干预依从性、方案偏离处理和主要比较策略。若为非劣效/等效研究，必须论证界值来源与临床合理性。
- 单臂或非随机干预研究：必须说明为何不采用随机设计，并补充历史对照选择、偏倚来源、时间趋势、混杂和解释边界。
- 观察性研究：应按 STROBE 思维预先布局；重点写清暴露定义、时间零点、结局来源、混杂因素、选择偏倚与信息偏倚控制。
- 诊断研究：应借鉴 STARD 逻辑进行 protocol 设计；重点写清目标用途（筛查/诊断/分层/疗效监测）、受试者连续入组、index test、reference standard、盲法、样本处理一致性、失败结果处理和临床应用场景。
- 预测模型研究：应在 protocol 阶段就区分开发、内部验证、外部验证和更新；避免把‘建模’写成黑箱。变量获取、缺失值处理、特征选择、过拟合控制、校准、判别、决策曲线和临床部署前提需预设。
- AI 研究：若干预或试验对象包含 AI 组件，protocol 应结合 SPIRIT-AI；应说明算法版本、输入数据、人工交互、异常输入处理、模型更新策略、人机责任边界以及现场可复现性。
- 以患者报告结局为主或关键次要终点的研究：应结合 SPIRIT-PRO；说明量表选择理由、语言版本、测量时间点、缺失 PRO 数据处理和临床解释。
- 终点复杂或容易选择性报告的研究：应结合 SPIRIT-Outcomes；把每个关键终点的定义、测量工具、时间点、聚合方法和解释框架写清。

## 12. 从评委、伦理和期刊角度看 protocol 的常见硬伤

- 问题不重要：研究主题存在，但不足以改变临床、指南或政策决策。
- 问题不清楚：研究目的过多、主次不分，导致样本量和统计分析都无法自洽。
- 设计不匹配：想证明因果却用弱设计；想做确认性结论却只有描述性数据基础。
- 对照无解释力：没有可比性，或选择会天然偏向试验组/暴露组的对照。
- 终点不合理：主要终点不临床、定义模糊、测量不可重复，或仅选 surrogate 却未解释其合理性。
- 样本量站不住脚：效应量来源虚弱，入组率明显高估，事件率和失访率不现实。
- 执行路径不闭环：没有 EDC、监查、质控、样本运输、影像判读或 AE 上报的可执行细节。
- 伦理风险高于潜在收益：风险控制、救济和隐私保护写得过于空泛。
- 写作方式像论文引言：背景很多，执行细节很少。
- 自动化改写痕迹明显：大量正确但空泛的话，缺乏可执行定义和本地化细节。

## 13. 建议的 Codex 自动化输出格式

- 第一部分：项目一句话定位（研究价值、目标人群、关键问题、设计类型）。
- 第二部分：主要缺陷清单（按价值定位、科学性、可行性、伦理性、写作完整性分层）。
- 第三部分：建议修改原则（先改逻辑，再改术语，再改格式）。
- 第四部分：可直接替换的正文草案（分章节输出，明确标注新增/修改内容）。
- 第五部分：仍需 PI 或合作方补充的信息清单（仅列缺失信息，不擅自编造）。
- 第六部分：研究类型专属提醒（RCT/观察性/诊断/预测模型/AI/实施研究）。
- 第七部分：最终核查表（注册、伦理、时间表、终点、样本量、统计、数据质量、安全性）。

## 14. 给 Codex 的硬约束

- 不得编造未提供的数字、机构、审批信息、注册信息、药品供应条件、前期数据或统计假设。
- 不得为了让文本显得完整而自动补写无法核实的效应量、失访率、事件率、AE 分级标准、检测阈值或中心数量。
- 不得把 exploratory 研究写成 confirmatory 研究；不得把单中心、有限样本、非随机设计包装成能直接改变指南的证据。
- 当研究价值不足时，应明确指出‘价值定位偏弱’，而不是只做语言润色。
- 当研究设计与目标不匹配时，应优先建议调整研究问题或设计，而不是硬性美化原方案。
- 所有需要研究者决策的地方，都应用标签标注，例如：[需PI确认]、[需补充先导数据]、[需统计师确认]、[需伦理确认]。
- 如需输出修改后的 protocol，必须保留版本号和修订日期，并建议生成 revision log。

## 15. 一页式快速审查清单

- 这项研究是否解决了一个重要且尚未被充分回答的问题？
- 一句话能否说清：谁、在什么场景、解决什么问题、用什么设计？
- 主要研究问题是否只有一个主轴？
- 研究设计是否足以回答该问题？
- 主要终点是否临床重要、定义清晰、可重复测量？
- 对照是否有解释力？
- 样本量假设是否来自可信依据？
- 招募、检测、随访、EDC、质控、安全上报是否真的能落地？
- 偏倚、缺失数据和 intercurrent events 是否考虑充分？
- 伦理与数据保护安排是否具体？
- 方案写完后，能否直接支撑伦理提交、注册和中心培训？
- 研究完成后，结果是否大概率能发表，并影响实际决策？
